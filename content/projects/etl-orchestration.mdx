## Executive Summary

Data pipelines are the lifeblood of analytics and machine learning operations. This project builds a production-grade orchestration framework that ingests data from 15+ sources, transforms it through multi-stage jobs, and loads it into a centralized data warehouse—all while handling failures gracefully and providing end-to-end visibility.

## System Architecture

The ETL system is built on Apache Airflow, a battle-tested workflow orchestration platform. The architecture follows a modular design:

**Ingestion Layer:** Custom connectors pull data from REST APIs, relational databases, cloud storage (S3), and event streams. Each connector implements exponential backoff and idempotent retries.

**Transformation Layer:** Spark jobs and Python scripts normalize, deduplicate, and enrich data. dbt handles data modeling and transformation logic in SQL.

**Loading Layer:** Transformed data lands in PostgreSQL data warehouse, partitioned by date and indexed for analytics queries.

## Implementation Details

### Airflow DAG Definition

```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime, timedelta

default_args = {
    'owner': 'data-engineering',
    'start_date': datetime(2025, 1, 1),
    'retries': 3,
    'retry_delay': timedelta(minutes=5),
}

dag = DAG(
    'daily_data_pipeline',
    default_args=default_args,
    schedule_interval='0 2 * * *',
)

def extract_api_data(**context):
    import requests
    endpoint = "https://api.example.com/events"
    response = requests.get(endpoint)
    data = response.json()
    return {'records_extracted': len(data)}

extract_task = PythonOperator(
    task_id='extract_api_data',
    python_callable=extract_api_data,
    dag=dag,
)
```
